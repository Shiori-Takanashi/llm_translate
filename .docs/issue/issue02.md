# Issue02: 設定初期化の責務を整理する

## 背景
現状、**presentation層**の`select_json()`において、
`load_env()`と`require()`を直に呼び出している。

`select_json()`はサブコマンドなので、
そこで設定初期化を行うべきではない。
責務が分散するからである。

## 目的
- `load_env()`はアプリ起動時に1回だけ実行する。
- `require()`の呼び出しを**境界**に集約する。
- applicationが担当するロジックが、環境に依存しないようにする。

## 方針
- 起動時に`AppConfig`を構築する。
- 各コマンドに設定を注入する。

## 完了条件
- `load_env()`がコマンド関数内から削除されている。
- アプリ起動時に**1回**のみ`load_env()`が実行される。
- application層は`require()`を呼ばない。
- `JSON_DIR`が未設定であるとき、エラーが明確に起きる。

## スコープ外
- `dotenv`を代替すること
