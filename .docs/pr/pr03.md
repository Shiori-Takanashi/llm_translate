# PR03: PROJECT_ROOT依存の解決（08fda30）

## 問題の概要

infra層がPROJECT_ROOT定数に直接依存していたため、
レイヤー分離およびテスト容易性の観点から問題があった。
その依存を除去する。

## infra/filesys.py

**変更前**
```python
from llm_translate.infra.root import PROJECT_ROOT
def listup_json(dir_name: str) -> list[str]:
    d_path = PROJECT_ROOT / dir_name
```

**変更後**
```python
`def listup_json(base_path: Path, dir_name: str) -> list[str]:
    d_path = base_path / dir_name
```

**説明**

- `listup_json()`にbase_pathをDIすることでPROJECT_ROOT定数を使用することを回避。
- これによりinfra層がグローバル状態に依存しない構造になる。
- `select_json()`が`list_json()`を呼び出すので、下に記す。

---

## presentation/cli/select_json.py

**変更前**

```python
@click.command(name="select-json")
@click.pass_obj
def select_json(config: AppConfig) -> None:
    dir_name = config.json_dir
    files = listup_json(dir_name)
```

**変更後**

```python
@click.command(name="select-json")
@click.pass_obj
def select_json(config: AppConfig) -> None:
    dir_name = config.json_dir
    base_path = config.base_path
    files = listup_json(base_path, dir_name)
```

**説明**

- 前提として`base_path`には実質的に**プロジェクトルートのPath**である。
- `listup_json()`の仕様変更に伴い、`base_path`を引数として指定。
- `base_path`をcli層で定義するのは責務の分散である。
- `AppConfig`を通して`base_path`を`main.py`から渡すことで責務を集約。
- これによりルート決定の責務をmainに限定できる。

---

# 影響範囲

- `listup_json()`のシグネチャ変更
- `select_json()`の修正

他モジュールへの影響は無し。

# 発見した問題点

## 返り値の不整合

`listup_json() -> list[str]`であるから、
選択したものを、後続する処理に使いにくい。
なぜなら、後続処理で再度`Path`に変換する必要があるから。
つまり、以下の設計原則に反している。

## 設計原則

- CLI層で抽象度が落ちている。
- Pathを扱う責務の境界が曖昧になっている。

## 解決策

Issue04として問題提起して、
`listup_json() -> list[Path]`に変更する。
